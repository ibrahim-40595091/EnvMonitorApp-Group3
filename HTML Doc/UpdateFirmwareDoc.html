Update Sensor Firmware

This documentation explains the implementation of the issue that allows users to specify a file path for uploading or referencing a firmware update for each sensor.

Overview

The goal of this issue is to prepare sensors for firmware updates by allowing users to manually provide a file path. This path could point to a local or remote file containing the update instructions or software.

Key Components

  • FilePathPage: A dedicated page that presents three input fields (one for each sensor type) where users can type or paste a file path that contains the required firmware update.
  • Sensor Classes: Each sensor class includes a FilePath property that stores the provided path as a string.
  • AppData: Holds all sensor instances in memory, making the file path accessible for later use when a firmware update is triggered.

User Flow

  • The user navigates to the HomePage after logging in.
  • The user navigates to the File Path Page via the Home Page.
  • Each section of the page contains an input field corresponding to one of the sensor types: air , water , or weather.
  • The user enters the file path where the firmware update file is located.
  • When the "Save File Paths" button is clicked:
    • The values entered are assigned to the FilePath property of each respective sensor.
    • A success message is shown confirming the values were saved.
    • The app returns to the Home Page.

Outcome

This issue provides the user with a simple interface to prepare sensors for firmware updates by associating them with a file path. If needed the system can then retrieve these filepaths to use when a firmware update is triggered
