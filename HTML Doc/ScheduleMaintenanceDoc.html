Schedule Maintenance for Sensors

This documentation outlines the implementation for the Issue that allows users to set a maintenance date for each sensor.

Overview

This feature provides a user interface that allows maintenance scheduling for all sensor types â€” air, water, and weather. It ensures that users can schedule a maintenance date and also there is a method that checks wether maintenance is due for a sensor today in the background.

Key Components

  • MaintenancePage: This page displays three date pickers, one for each sensor type, and a button labeled "Apply".
  • Sensor Classes: Each sensor type includes a MaintenanceDate property which stores the selected date.
  • AppData: Maintains a shared collection of sensor objects, allowing the maintenance dates to be updated globally.

User Flow

  • The user logins from the LoginPage.
  • The user navigates to the Maintenance Page via the Home Page.
  • The page displays one date picker for each sensor type.
  • The user selects a desired maintenance date for each sensor.
  • Upon clicking the "Apply" button:
    • The chosen dates are saved to the corresponding sensors.
    • A confirmation message is shown to the user.
    • The app navigates back to the Home Page.

Outcome

Users can plan maintenance for the sensors well in advance, makig sure sensors stay working and also checks to see if a sensor is due for maintenance so that the user is always alerted.
